## Milestone 7c ‚Äî Hero % count-up on first load ‚Äî Tests

This entry covers M7c ‚Äî the third item in the Polish Layer: a 1.6 s easeOut tween of the hero's `0%` ‚Üí `dayPct(state)`'s integer percent, fired exactly once on first hydrated paint of `<BuildingClient>`. Both the SVG `stroke-dashoffset` and the visible 72-px numeral ride a single shared `displayPct` React state inside `<HeroRing>` (joint-state per AC #1 ‚Äî they CANNOT desync). Render-layer only ‚Äî no schema bump, no new persisted field, no new `Action` union member, no new clock, no new CSS custom property. M7c reuses M7a's `useFirstPaintAfterHydration(hydrated)` ref machine verbatim (the same `stagger` boolean at `BuildingClient` line 177 is threaded to `<Hero firstPaintCountUp={stagger}>`) ‚Äî **no parallel ref-machine** (explicit dispatch-prompt mandate; plan ¬ß Decisions honored anchors it as the PRIMARY ADR-023 honor). M7c adds ONE additive motion token (`motionTokens.countUp = { durationMs: 1600, easing: "easeOut" }`) in `lib/motion.ts`, ONE optional prop `firstPaintCountUp?: boolean` on `<HeroRing>` + `<Hero>` (default `false` ‚Äî backwards-compat), and ONE additive `children` type-union extension on `<HeroRing>` (`ReactNode | (displayPct: number) => ReactNode`). It is derived from the `plan.md` M7c entry (`## Milestone 7c ‚Äî Hero % count-up on first load ‚Äî Plan`, commit `1d15fba`). Feature slug: `m7c`. ID prefixes: `U-m7c-`, `C-m7c-`, `E-m7c-`, `A-m7c-`. M7c has a tiny pure-logic surface (the `motionTokens.countUp` token + `getMotion("countUp", ‚Ä¶)` exhaustiveness), a real component surface (`<HeroRing>` tween effect + joint-state `displayPct` driving stroke AND numeral + fire-once-per-mount contract + PRM collapse + children-as-function; `<Hero>` pass-through; `<BuildingClient>` one-line `firstPaintCountUp={stagger}` thread), and a deferred-to-preview perf + a11y surface (Lighthouse Perf ‚â• 90; count-up completes inside 1.7 s; stroke ‚Üî numeral sync to within one frame; zero CLS; axe-clean during and after the tween), so it has the full four-layer test mix ‚Äî unit (Vitest), component (Vitest + Testing Library), accessibility (axe via Playwright, deferred-to-preview), and E2E (Playwright, deferred-to-preview).
